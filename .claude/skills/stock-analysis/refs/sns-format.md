# SNS Content Format

`snsContent` 필드를 리포트 JSON에 추가한다.

**핵심 원칙: `overallOpinion`이 곧 SNS 콘텐츠다.** 별도로 새 글을 쓰지 않고, overallOpinion을 플랫폼 포맷에 맞게 나누기만 한다.

## 변환 규칙

1. overallOpinion에서 가장 임팩트 있는 문장 → 첫 트윗(훅)
2. 나머지 항목 → 하나씩 트윗으로 분리
3. 마지막 트윗 → 목표가, 현재 위치, 결론

## 글쓰기 스타일

투자 커뮤니티에서 신뢰받는 사람 톤. 데이터는 정확, 표현은 편하게.
- 보고서 톤/과한 이모지/감탄사 금지
- 짧은 문장, 한 문장에 한 정보
- 주어 명확히: "업그레이드됐다"(X) → "Baird가 업그레이드했다"(O)
- em dash, 가운뎃점 절대 금지. 콤마, 마침표, 괄호, 콜론 사용
- 링크 넣지 마라 (공유 버튼이 자동 추가)

## 플랫폼별 포맷

### X (Twitter)
- 트윗당 280자 이하, 스레드 번호(`\n\n1/4`) 필수, 3-4개 적정(최대 5개)
- 예시:
```
text: "$TSLA (3/7) 자율주행 심사가 이틀 남았다. 통과하면 반등, 실패하면 30% 급락. $396.73 마감.\n\n1/4"
thread: ["영국 2월 판매 37% 급감, Toyota 탄소 협력 탈퇴. 유럽 전망 어두워졌다.\n\n2/4", ...]
```

### Threads
- 글자 제한 없음. 단일 `text`. 티커+가격 → 본문 2-3단락 → 결론

### Telegram
- 단일 `text`. 구조화된 섹션(핵심 이슈, 주가 위치, 목표가)

## JSON 구조

```jsonc
"snsContent": {
  "x": { "hook": "한 줄 요약", "text": "첫 트윗\n\n1/N", "thread": ["2/N", ...] },
  "threads": { "hook": "한 줄 요약", "text": "전체 본문" },
  "telegram": { "hook": "한 줄 요약", "text": "구조화된 본문" },
  "tones": {
    "x": {
      "witty":   { "hook": "...", "text": "...", "thread": ["..."] },
      "empathy": { "hook": "...", "text": "...", "thread": ["..."] }
    },
    "threads": {
      "witty":   { "hook": "...", "text": "..." },
      "empathy": { "hook": "...", "text": "..." }
    }
  }
}
```

**fact 톤은 생성하지 않는다.** 기본 `snsContent.x` = fact 톤이므로 중복.

## 톤 변주 (witty, empathy만)

| Tone | 스타일 |
|------|--------|
| `witty` | 대비/아이러니로 재미. 웃기면서도 투자 정보 포함 |
| `empathy` | 주주 감정 인정, 좋은/나쁜 소식 솔직 전달, fact보다 길어도 OK |

모든 톤에 핵심 데이터(가격, 목표가, 상승여력%) 포함 필수.
